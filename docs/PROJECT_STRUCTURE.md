# Project Structure

## Root

- `README.md`: public project overview.
- `ORBITGUARD_REQUIREMENTS.md`: product requirements.
- `HLD.md`: system architecture and testing architecture.
- `METHODOLOGY.md`: science and assumptions.
- `DEMO_SCRIPT.md`: video and live-demo flow.
- `FINAL_ROUND_ROADMAP.md`: post-Round 1 roadmap.

## Engineering Documentation

- `docs/`: onboarding, project rules, feature-change process, and structure guide.
- `architecture/`: LLD for each service/feature.
- `context/`: current state and per-module handoff context.
- `decisions/`: durable architecture decision records.

## Planning And Submission

- `execution/`: build order, testing checklist, submission checklist.
- `demo/`: video, deck, and rubric mapping.
- `research/`: science, data, ISRO, and competitor notes.
- `data-plan/`: data snapshot and offline demo plans.

## Code

- `backend/`: FastAPI backend and Python domain services.
- `frontend/`: React/Vite/TypeScript mission console with Three.js visualization.
- `data/`: committed deterministic fixtures, catalog snapshots, expected demo flows, and secondary-risk fixtures.
- `scripts/`: future automation scripts.

## Rule For New Folders

Add a `README.md` to every new top-level folder explaining purpose, ownership, and how it should be used.
