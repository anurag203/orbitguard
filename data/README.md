# OrbitGuard Data

This folder contains deterministic data needed for offline demos and tests.

## Rules

- Demo-critical data must be committed.
- Live refresh scripts may update snapshots later, but the Protect ISRO hero flow must not require live internet.
- Every dataset must describe its source, snapshot status, and limitations.
- Simulated or injected events must be labeled clearly.

## Folders

- `catalogs/`: catalog snapshots and watchlists.
